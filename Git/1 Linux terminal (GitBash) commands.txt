Linux terminal (GitBash) commands

1) Посмотреть где я - pwd
2) Создать папку - mkdir dir (убедиться, что создалась папка ls -la)
3) Зайти в папку - cd ./dir
4) Создать 3 папки - mkdir {d1,d2,d3} (убедиться ls -la)
5) Зайти в любую папку - cd ./d1
6) Создать 5 файлов (3 txt, 2 json) - touch {1.txt,2.txt,3.txt,4.json,5.json} (или по одному файлу командой > 1.txt и т.д.)
7) Создать 3 папки - mkdir {d4,d5,d6}
8. Вывести список содержимого папки - ls -la
9) + Открыть любой txt файл - vim 1.txt (или nano 1.txt)
10) + написать туда что-нибудь, любой текст. - (просто пишем текст)
11) + сохранить и выйти. - если vim, то esc :wq (если nano, то ctrl+X -> Y (чтобы сохранить изменения)
12) Выйти из папки на уровень выше - cd ..
—
13) переместить любые 2 файла, которые вы создали, в любую другую папку. - mv ./d1/1.txt ./d1/2.txt ./d2/d4
14) скопировать любые 2 файла, которые вы создали, в любую другую папку. - cp ./d1/1.txt ./d1/2.txt ./d1/d4/
15) Найти файл по имени - find -name 4.*
16) просмотреть содержимое в реальном времени (команда grep) изучите как она работает. - просмотреть командой grep строку: grep был ./d1/9.txt 
выдать содержимое в реальном времени файла не grep: tail -f ./d1/9.txt
выдать строку в режиме реального времени tail -f ./d1/9.txt | grep был
17) вывести несколько первых строк из текстового файла head -7 9.txt или head -n 7 9.txt
18) вывести несколько последних строк из текстового файла - tail -7 9.txt
19) просмотреть содержимое длинного файла (команда less) изучите как она работает. - less 9.txt
20) вывести дату и время - date
