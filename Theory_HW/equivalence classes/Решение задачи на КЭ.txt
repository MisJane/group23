Разработана Блок-схема со всеми вариантами, подготовлены данные, разбитые по КЭ:

Будет удобнее выделить пять классов эквивалентности:
	1. Пользователь — не участник бонусной программы. Все остальные условия в этом случае не имеют значения, результат всегда один — скидки нет. Чтобы проверить этот класс эквивалентности, достаточно выполнить одно условие — подобрать пользователя без бонусной программы, а для остальных условий использовать любые данные.
	2. Пользователь — участник бонусной программы, стоимость товара меньше 1000 рублей, бейджа на товаре нет. Скидка составит 5%.
	3. Пользователь — участник бонусной программы, стоимость товара меньше 1000 рублей, бейдж на товаре есть. Скидка составит 12%.
	4. Пользователь — участник бонусной программы, стоимость товара больше 1000 рублей, бейджа на товаре нет. Скидка составит 10%.
	5. Пользователь — участник бонусной программы, стоимость товара больше 1000 рублей, бейдж на товаре есть. Скидка составит 17%.

Для каждого класса эквивалентности нужно подобрать данные и выполнить по одному тесту.

Если не выделять классы, а проверять каждое требование по отдельности, можно ошибиться: упустить или продублировать часть проверок.
